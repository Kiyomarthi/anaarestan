# قوانین

1. ابتدا فایل AI\architecture.md رو بخون این فایل شامل معماری و ساختار پروژه می باشد
2. پروژه , typscript, javascript, nuxt4 , nuxt ui, tailwind و ... در آخرین ورژن استفاده شده اند
3. حتما برای اینکه بتونی با دیتا بیس کار کنی کوئری بزن و جدول های لازم رو نگاه بنداز و ساختار دیتابیس رو بشناس و در صورت نیاز چنانچه api نیاز به جدول جدید داشت آن را درست کن
4. این فایل رو حتما بخون: AI\design-guidline.md
5. همچنین برای آشنا شدن با فعالیت هایی که انجام شده این فولدر رو نیز بخون: openspec
6. در تمامی پروژه تایپ اسکریپت رو حتما رعایت کن و تایپ ها رو کامل بگذار
7. به هیچ عنوان از این استفاده نکن: app\composables\useApiFetch.ts
8. در صورت نیاز موارد .env و مشابه آن را بخون و اگه نیاز است که مورد جدیدی اضافه کنی حتما اضافه کن و حتما در nuxt.config.ts در قسمت  runtimeConfig: { باید اضافه کنی و از این تیکه runtime در تمامی پروژه اضافه کنی به هیچ وجه در هیج جایی از پروژه نباید مستقیما .env رو بخونی بلکه باید از runtime بخونی
9. در صورت نیاز این فایل رو خودت آپدیت کن و موارد جدید رو اضافه کن
10. جهت api زدن قسمت هایی که ssr می باشد از این کامپوزبل : app\composables\useCacheFetch.ts و برای کلاینت ها نیز از این کامپوزبل: app\composables\useApiRequest.ts به هیچ عنوان برای api زدن از غیر اینها نباید استفاده کنی مخصوص از این نباید استفاده کنی: app\composables\useApiFetch.ts این رو هیچ وقت استفاده نکن همچنین از آنجایی که تمامی موارد لازم از جمله loading, data, ... را خود این کامپوزبل ها در نظر گرفتن نیاز نیست هر سری مجدد خودت بنویسی صرفا مطالعه کن و استفاده کن نیازی نیست که از try , catche استفاده کنی چون خود اینها استفاده کردن به هیچ عنوان اضافه ننویس
11. شما باید از کامپوننت های nuxt ui استفاده کنی به این معنی که تا حد خیلی زیادی نیاز نیست که در app\components\base کامپوننت درست کنی چرا که تمامی کامپونننت بیس های مورد نیاز در nuxt ui پیدا میشه پس به هیچ وجه نباید کامپوننتی رو بزنی که معادل اون در nuxt ui وجود داره: https://ui.nuxt.com/docs/components
12. وقتی که بهت دستور دادم که یک ویژگی جدید رو بزنی بنابر اون اسمی که بهت می دم باید بری فولدر: AI\prod و بگیر طبق openspec رفتار کن و در openspec\changes\archive فایل های لازم رو بنویس و تسک ها رو مشخص کن و قبل از اجرا از من تایید بگیر تا من فایل task.md اون prd رو تایید بدم بعد اون تسک ها رو انجام بدی
