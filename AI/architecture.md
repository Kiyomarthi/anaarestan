# معماری و ساختار پروژه

## فرانت اند (./app)

پروژه , typscript, javascript, nuxt4 , nuxt ui, tailwind و ...
همگی در آخرین ورژن خود استفاده شده اند

### components

app\components
تمامی کامپوننت های قابل استفاده در این فولدر نوشته می شوند
معماری به صورت اتمیک دیزاین پیش می رو

#### app\components\base اتم ها

1. به صورت پیش فرض شما باید از کامپوننت های nuxt ui استفاده کنی به این معنی که تا حد خیلی زیادی نیاز نیست که اینجا کامپوننت درست کنی چرا که تمامی کامپونننت بیس های مورد نیاز در nuxt ui پیدا میشه پس به هیچ وجه نباید کامپوننتی رو بزنی که معادل اون در nuxt ui وجود داره: https://ui.nuxt.com/docs/components
2. بیس ترین کامپوننت ها باید در این فولدر باشند: app\components\base کامپوننت های اتم شده در این فولدر هستند
3. هیچ مقداری به صورت استاتیک نباید باشه و باید مقادیر و تعاملت به صورت props, emit, v-model باشد
4. در رابطه با کلاس ها و استایل هایی که داده می شود نیز در صورتی که نیاز باشد در جاهای مختلف کلاس های مختلفی اعمال بشه باید از این استفاده بشه :

```
const props = withDefaults(

  defineProps<{

    class?: string;



  {

    class: "",

  }

);

const imageClasses = computed(() =>

  twMerge("w-full h-full object-cover", props.imageClass)

);

<nuxt-img

      :class="imageClasses"

    />
   
   
```

این نحوه کلاس دهی رعایت شود. 5. اسم گذاری و پوشه بندی به این صورت است که باید مفهمومی و معنایی باشد مثلا اگر یک کامپوننت داریم که محتوا را در دسکتاپ مدال می کند و در موبایل اسلاید اور می کند این اسم: response-modal و همچنین چنانچه به هم مرتبط هستن داخل یک فولدر قرار می گیرند: مثلا من چند مدل دکمه دارم یک فولدر به نام button و داخل آن fill, outline , .... soft, ... برای مثال به این گونه باشد و به صورت پیش فرض تمامی کامپوننت ها باید اسمشان فولدر باشد و خود کامپوننت داخل آن: index.vue به این گونه یعنی مثلا پوشه buttom داخل آن: index.vue

3.  هیچ مقداری به صورت استاتیک نباید باشه و باید مقادیر و تعاملت به صورت props, emit, v-model باشد

#### model, مدل ها, app\components\model

1. اینجا شما از کامپوننت های آماده nuxt ui و widget های ساخته شده استفاده می کنی
2. api را اینجا فراخونی می کنی و منطق های خیلی پیچیده که برای سناریو های خاص هستن وتکرار نمی شوند را در اینجا پیاده سازی می کنی
3. اسم گذاری و پوشه بندی به این صورت است که باید مفهمومی و معنایی باشد مثلا ما یک مدل داریم به نام product => این یک فولدر می شود و داخل آن : cart, list, counter, ...یا مدل dashboard که داخل آن profile, address, transactions, wallet, likes, ... همه چی یک فولدر دارد حتی اگه یک مورد باشد باز هم فولدر که داخل ان: index.vue
4. جهت api زدن قسمت هایی که ssr می باشد از این کامپوزبل : app\composables\useCacheFetch.ts و برای کلاینت ها نیز از این کامپوزبل: app\composables\useApiRequest.ts به هیچ عنوان برای api زدن از غیر اینها نباید استفاده کنی مخصوص از این نباید استفاده کنی: app\composables\useApiFetch.ts این رو هیچ وقت استفاده نکن همچنین از آنجایی که تمامی موارد لازم از جمله loading, data, ... را خود این کامپوزبل ها در نظر گرفتن نیاز نیست هر سری مجدد خودت بنویسی صرفا مطالعه کن و استفاده کن نیازی نیست که از try , catche استفاده کنی چون خود اینها استفاده کردن به هیچ عنوان اضافه ننویس مثال

```
await getGallery("/api/upload/arvan", {

    headers: {

      Authorization: token,

    },

  });
```

به همین اندازه که استفاده کنی کاملا کافیه از نوشتن موارد غیر لازم پرهیز کن 5. حتما وقتی که ظاهر می زنی ui می زنی دقت کن که جاهایی که لازم است باید با استفاده از USkeleton در nuxt ui حتما لودر هاش رو هم تنظیم و درست بکنی

## بکند server

1.  در آدرس /server تمامی قسمت های مربوط به بکند باید پیاده سازی شود

### server\api

1. تمامی api های لازم در اینجا پیاده سازی می شود ساختار رو حفظ کن
2. به این صورت نوشته می شود مثلا اگه مدل product داریم یک پوشه به این نام نوشته شده و داخل آن: index.get.ts, index.post.ts, [code].get.ts, [code].path.ts, [code].delete.ts index.delete.ts خلاصه که باید کراد به صورت کامل پیاده سازی شود
3. مثل مابقی api هایی که زده شده باید همون الگو رو رعایت کنی
4. برای ولیدیت کردن دیتا از این استفاده می کنی:

```
 validateBody(body, {

    user_id: (v) => validate(v).run(),

    status: (v) =>

      v === undefined

        ? true

        : validate(v).checkMatch(["active", "converted", "abandoned"]).run(),

  });
```

در صورتی که ولدیت خاصی رو نتونستی پیدا کنی باید برای اون بسازی و استفاده کنی: shared\validation\index.ts در این آدرس رولز مورد نظر رو می سازی یا برحسب نیاز آپدیت می کنی و استفاده می کنی و از نوشتن کد های سخت تا جای ممکن جلوگیری می کنی

### server\db\index.ts

1. این کوئری اتصال به دیتابیس می باشد برای اینکه نیاز داشتی در api بگیری کوئری بزنی به دیتا بیس از این استفاده می کنی
2. در صورتی که جدول مورد نظر نبود از همین استفاده می کنی برای اینکه کوئری بزنی و یک جدول جدید درست حذف و یا تغییر بدی

### server\utils

شامل utils های مخصوص سمت server می باشد در صورتی که فانکشنی داری که مطمئن هستی به هیچ وجه سمت فرانت اند استفاده نمی شود و مخصوص بکند است در این جا درست کن

## shared

این قمست شامل تایپ های تایپ اسکریپت shared\types و همچنین shared\utils ها می باشد که در هم سمت سرور و هم فرانت اند استفاده می شود

1. زمانی که داری یک api جدید می نویسی باید تایپ های لازم اون رو هم حتما بسازی تا چه درسمت بکند و فرانت بدون مشکل استفاده بشه
2. فانکشن هایی که چندین بار استفاده می شن رو حتما در اینجا بنویس: shared\utils

موارد مهم:

1. در تمامی پروژه تایپ اسکریپت رو حتما رعایت کن و تایپ ها رو کامل بگذار
2. به هیچ عنوان از این استفاده نکن: app\composables\useApiFetch.ts
3. در صورت نیاز موارد .env و مشابه آن را بخون و اگه نیاز است که مورد جدیدی اضافه کنی حتما اضافه کن و حتما در nuxt.config.ts در قسمت  runtimeConfig: { باید اضافه کنی و از این تیکه runtime در تمامی پروژه اضافه کنی به هیچ وجه در هیج جایی از پروژه نباید مستقیما .env رو بخونی بلکه باید از runtime بخونی
4. در صورت نیاز این فایل رو خودت آپدیت کن و موارد جدید رو اضافه کن
